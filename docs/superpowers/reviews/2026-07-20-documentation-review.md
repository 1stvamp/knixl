# Documentation review and usage media (2026-07-20)

A pass over the whole doc set (README, docs/00-06, docs/adr/0001-0007, examples/README) plus
new usage media. The review ran as four subagents with different lenses: accuracy against the
code, structure and coverage, new-user onboarding, and ADR consistency plus house style. Clear
factual and style corrections were applied on this branch; the judgement calls below are left
for a decision.

## Judgement calls (left for a decision)

These are structure and content opinions, not factual gaps. Nothing here is applied.

- **Split `install` into its own chapter.** The pinning, strategy selection, and per-host
  baseline material now lives in an ever-growing `install` section in docs/05. A dedicated
  `docs/07-package-pinning.md` would keep docs/05 a terse command reference. Recommendation:
  worth doing given three ADRs (0005/0006/0007) now land on `install`.
- **docs/00 orientation.** docs/00 is prior-art only. The README now carries onboarding
  (quickstart, glossary), so docs/00 can stay as the competitive write-up, or gain a short
  getting-started. Recommendation: leave docs/00 as-is now that the README onboards; revisit if
  a standalone getting-started chapter is wanted.
- **Author-a-module tutorial.** docs/04 is a grammar reference, not a task walkthrough. A short
  tutorial taking `security-headers` from schema to working host output would help module authors.
- **ADR vs numbered-doc duplication.** ADRs 0001-0004 restate mechanism that also lives in the
  numbered docs; 0005-0007 did not (now partly fixed by the docs/05/06 additions above). Worth
  picking one convention: ADRs as terse decision logs pointing at the numbered docs for "how".
- **Spec-to-docs promotion.** Shipped specs under `docs/superpowers/specs/` were never folded
  into the numbered docs, which is how the docs fell behind. A line in the finishing-a-feature
  workflow (update the relevant docs/0X chapter on spec close-out) would stop the drift.
- **Nice-to-haves.** A `knixl init` project scaffold (a feature, not docs); a CONTRIBUTING note
  pointing at `mise.toml`'s tasks; an annotated walk-through of one real lock entry.
